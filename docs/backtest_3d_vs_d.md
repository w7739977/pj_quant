# 3d 方案 vs d 方案回测对照 — 阶段 1 信号质量评估

**生成日期**: 2026-05-11
**回测区间**: 2025-01-03 ~ 2026-04-27 (约 16 个月 / 62-69 周)
**模型**: `xgb_ranker.json` (PRODUCTION_MODEL)
**池**: 5-50 亿小盘股 (~1700 只)
**框架层**: L2 (信号质量评估 — 见 `VALIDATION_PHILOSOPHY.md`)

> **⚠️ 信号视角，不是实盘 PnL**：每次推送独立计 5d 收益，不模拟持仓重叠 / 换仓决策 / 交易成本。检验的是"频次升高对模型选股信号质量的影响"。生产实盘要看的是 L3 模拟交易（本报告不涉及）。

## 唯一变量

| 维度 | d 方案 (生产) | 3d 方案 (新提案) |
|---|---|---|
| **推送频次** | 每周一 | **每周一、三、五** |
| 每次推 | top 10 | top 10 |
| Hold | 5 工作日 | 5 工作日 |
| Window | 5 工作日 cache | 5 工作日 cache |
| 节假日 | 跳过 | 跳过 |

## 总览对照（按周聚合）

| 指标 | d (周一) | 3d (周一/三/五) | 差异 | 怎么读 |
|---|---:|---:|---:|---|
| 总周数 | 62 | 69 | +7 | 3d 多 7 周是因为有些周只有周三/周五推（周一假期）|
| picks 总数 | 614 | 1883 | ×3.1 | 3d 推送频次是 d 的 ~3 倍 |
| **胜率** | **54.1%** | **55.1%** | **+1.1pp** | 几乎一样（50% 是 coin flip） |
| 平均 5d α | +0.89pp | +0.91pp | **+0.02pp** | 几乎一样 |
| **Profit Factor** | **1.79** | **1.81** | **+0.02** | 几乎一样（业界 1.5 合格 2.0 优秀）|
| Expectancy (单只) | +1.44% | +1.44% | 0 | 完全一样 |
| 累计 α (周聚合复利) | **+73.44%** | **+88.07%** | +14.6pp | 见下方注释 |
| 周度 α σ | 1.99% | 2.05% | — | 波动性几乎一样 |

> **累计 α 注释**：3d 的 +88% 看似比 d 的 +73% 高 +15pp。但**统计上不显著**（见下方 paired t test）。3d 多 7 周（周一假期时仍有周三/周五）也贡献部分差异。

> **修复了之前的 artifact**：早期版本累计 α 算成 +440% 是个 bug（按 push_date 而非 week 累乘 cumprod），实际正确算法是 weekly cumprod。

## 排除 ST 后对照（真 alpha 视角）

| 指标 | d 排 ST | 3d 排 ST | 差异 |
|---|---:|---:|---:|
| picks 数 | 487 | 1490 | ×3 |
| 胜率 | 52.2% | 53.8% | +1.6pp |
| 平均 5d α | +0.38pp | +0.39pp | **+0.01pp** |
| Profit Factor | 1.44 | 1.50 | +0.07 |
| 累计 α (周聚合) | **+10.05%** | **+14.82%** | +4.8pp |

**关键发现**：排 ST 后真 alpha 极弱。62 周年化仅 **+7-8%**（L2 假设性，扣实盘成本后会更弱）。这跟之前 14 周 picks_performance_no_st.md 的结论一致 — 模型主要 alpha 来自 ST 反弹。

## 配对 t 检验（核心统计判断）

把 d 和 3d 在**同周**的 weekly α 配对 (62 个共同周)：

```
3d - d 周度 α 差均值: +0.095 pp
σ:                    1.519%
SE:                   0.193%
t-statistic:          0.494
p-value (双尾):       0.6233
```

**判断**：**p = 0.62 远超 0.05 显著阈值** — 不能拒绝零假设「3d 和 d 信号质量相同」。

**62 周这么长的样本下仍达不到 0.05 显著** → 3d 频次升高的"额外信号机会"对模型的 alpha **没有实质提升**。

## 结论

### 信号层（L2）发现

1. **3d 在信号质量上跟 d 没有显著差异**（胜率 +1.1pp、PF +0.02、avg α +0.02pp 都在噪声范围内；paired t p=0.62）
2. **累计 α 表面差 +15pp** 是更多推送 + 偶尔多 7 周的统计 artifact，不是真 edge
3. **真 alpha 主要靠 ST**（排 ST 后 d / 3d 都只 +10-15%/年，扣成本接近 0）

### 实施建议

| 业务诉求 | 推荐 |
|---|---|
| **目的是「信号推送 / 信息提供」给客户** | ✅ 可以做 3d，工作量小（cron + 脚本改 ~30 min）。**但不要承诺累计 α 更高** |
| **目的是「换仓策略 PnL 更高」** | ❌ 信号层无 edge → L3 模拟交易也大概率无 edge；先解决 ST 反弹问题再说 |
| **目的是「累积样本验证更快」** | ✅ 3d 一周 3 个推送事件，累积到 80+ 显著样本时间从 80 周缩到 ~30 周（注意相关性让有效自由度 < 3 倍） |

### 严格按 VALIDATION_PHILOSOPHY.md

- **L2 没有显著优势 → L3 模拟交易没必要做**（之前预估的阶段 2 工作量 2.5-3.5h 可省）
- **如果业务一定要 3d 推送**：本报告可作为「信号层不优」的免责依据
- **真正应该优化的方向**：解决 ST 反弹问题，让排 ST 后真 alpha 升高（task #21 / #13 后续）

## 数据可重现

```bash
# 重跑回测
python3 scripts/backtest_3d_vs_d.py --start 2025-01-01

# 重算 weekly cumprod + 配对 t test (基于已有 csv)
python3 -c "
import pandas as pd
df = pd.read_csv('logs/backtest_3d_vs_d.csv')
df['push_date'] = pd.to_datetime(df['push_date'])
df['week'] = df['push_date'].dt.to_period('W').astype(str)
for method in ('d', '3d'):
    sub = df[df['method'] == method]
    weekly = sub.groupby('week')['alpha'].mean()
    cum = ((1 + weekly).cumprod() - 1).iloc[-1]
    print(f'{method}: weeks={len(weekly)}, picks={len(sub)}, cum_alpha={cum*100:+.2f}%')
"
```
