## 核心问题

"模型推的股票好不好？" 这个问题有 **多层含义**。把它们混淆是最常见的犯错。

## 四层验证框架 (L1 → L4)

| 层级 | 名称 | 测什么 | 衡量指标 | 跟"实盘真赚多少"的距离 |
|---|---|---|---|---|
| **L1** | 模型预测能力 | 模型对 forward return 的拟合度 | R² / IC / Rank IC | 最远 |
| **L2** | 信号选股质量 | 推荐 top N 的 forward return 平均涨跌 | 5d 胜率 / 5d α / Profit Factor | 中 |
| **L3** | 实盘策略 PnL | 含成本/滑点/止损规则的累积 PnL | 净累计 PnL / sharpe / max DD | 近 |
| **L4** | 真金白银 | 真账户净值变化 | 账户余额 | = |

### 单调约束

- **L2 好 ≠ L4 必然赚钱**（成本 / 滑点 / 流动性 / 涨跌停 / T+1 都可能吃掉信号）
- **L2 差 → L4 必然不赚钱**（信号本身没 edge，加什么策略也救不了）

**所以 L2 评估是"必要不充分"条件**。

## L2 评估的边界（写所有报告时必须说明）

picks_history / picks_performance 等所有报告的数字基于：

```
假设性 5d hold:
  推荐日 D 当天 close 价 = 买入价 (假设可成交)
  D + 5 工作日 close 价 = 卖出价 (假设可成交)
  ret_5d = (卖出价 - 买入价) / 买入价
```

**不包含**：
- ❌ 双边换手成本（A 股约 0.4-0.7%/单边）
- ❌ 滑点（小盘股 close → 实际成交价差异）
- ❌ ST 涨跌停 ±5% 限制（实际可能卖不出）
- ❌ 停牌 / 流动性不足
- ❌ T+1 当日买不能卖
- ❌ 生产实盘的动态 hold 规则（止损 -8% / 止盈 +15% / 20d 超时盘整）
- ❌ 持仓重叠（3d 方案三次推送 picks 重复入选问题）

**所以**：
- L2 累计 α = 「假设你能按 close 自由买卖、不付任何成本」的理论 α
- 实盘真实 PnL ≠ L2 累计 α
- L2 报告关注**信号是否有 edge**，不关注"跟着买能赚多少"

## 任何新策略都要走这条路径

```
L1 模型训练 → cv_r2 > 0.05 ✓ 
   ↓
L2 信号 forward OOS 评估 → 5d α > 0 且统计显著 ✓ (80+ 周观测)
   ↓
L3 模拟交易引擎 → 含成本/动态hold 模拟后净累计 > 0 ✓
   ↓
L4 小资金试运行 → 实盘 3-6 月 PnL 正向 ✓
   ↓
扩规模 / 加杠杆
```

**禁止跳级**：L2 数字漂亮就上 L4，是 quant 最常见的爆仓套路。

## 几个常见混淆

### 误区 1：「累计 α +27%」 = 「跟着买能赚 27%」

**错**。L2 累计 α 是假设性数字。真实 PnL 要扣：
- 成本 ~0.5%/单边 × 换手频次
- 滑点 ~0.1-0.3%/单边
- 实盘动态 hold 让你不一定能 hold 满 5d

实际 L3/L4 收益常常是 L2 累计 α 的 **30-60%**（业界经验：sharpe decay 30-60%）。

### 误区 2：「我没真买，怎么算胜率」

**不需要真买**。胜率 = 「假设性持有 5d 后涨跌为正的占比」。这是信号 IC 的代理指标，**测的是模型预测能力**，不是实盘表现。

### 误区 3：「3d 方案每次推 10 只，跟 d 推 10 只对比一周 30 vs 10 只，不公平」

不对。**对比是事件级**（每个推送日的 10 只独立评估），不是周级。3d 在一周内多了 2 个推送机会，每个机会都是独立 5d 评估。统计上 3d 样本量更大（约 3 倍），但**相邻推送的 picks 高度相关**（同一只股可能在周一和周三都进 top 10，5d 评估窗口重叠），所以**有效自由度提升 < 3 倍**。

## 实操约束（防止跳级）

1. **所有 backtest/picks_history 类报告**头部必须有「⚠️ 信号视角，不是实盘 PnL」免责块
2. **L2 数字单独看不下决策**：永远要伴随「样本是否足够（≥ 80 周）」「排 ST 后是否还成立」两个交叉验证
3. **改生产策略**（如 NUM_POSITIONS / hold 期 / 推送频次）前，必须在 L2 看到显著优势（paired t p < 0.05 或 DSR > 0.95）
4. **上真账户**前必须先做 L3 模拟交易 + 至少 6 个月 paper trading

## 给后续合作 / review 的判断标准

读到任何"alpha +X%"数字时，先问：
1. 这是 L1 / L2 / L3 / L4 的？
2. 假设条件是什么（hold 期 / 成本 / 池）？
3. 样本量够吗（周观测 ≥ 80）？
4. 排 ST / 排涨跌停 / 排停牌后还成立吗？

回答不上来 → 那个数字不能信。
