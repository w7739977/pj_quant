# A 股量化交易系统 — 选股与操作逻辑说明

## 整体架构

```
每日 15:30 收盘后自动触发:
  增量数据拉取 → 健康检查 → 生成操作建议推送微信 → 归档信号
```

当前使用激进模式: 100% 资金投个股, 3 只集中持仓, 不配 ETF。

---

## 一、选股流程

### Step 1: 股票池筛选

从全 A 股 ~4800 只中, 筛选**市值 5 亿 ~ 100 亿**的小盘股, 排除 ST、停牌、科创板(688)、北交所。

### Step 2: 计算 15 个因子

每个因子在全截面做分位数排名(0~1), 按方向调整后加权求和, 得到**多因子综合分**。

| 类别 | 因子 | 含义 | 方向 | 权重 |
|---|---|---|---|---|
| **动量** | mom_5d, mom_10d, mom_20d, mom_60d | 过去 N 日涨幅 | 涨多好 | 2x |
| **波动** | vol_10d, vol_20d | 过去 N 日波动率 | 低波好 | 1x |
| **换手** | avg_turnover_5d, avg_turnover_20d, turnover_accel | 平均换手率 + 加速度 | 活跃好 | 1x |
| **量价** | vol_price_diverge, volume_surge | 量价背离 + 放量信号 | 放量好 | 1x |
| **技术** | ma5_bias, ma10_bias, rsi_14 | 均线偏离 + 相对强弱 | 超卖好 | 1x |
| **估值** | pe_ttm, pb | 市盈率 / 市净率 | 低估好 | 1.5x |
| **活跃** | turnover_rate, volume_ratio | 换手率 / 量比 | 活跃好 | 1x |

### Step 3: ML 模型预测

XGBoost 回归模型, 输入同样的 15 个因子 + 情绪分, 输出**未来 20 日预测收益率**, 按预测值排名得到 **ML 排名**。(详见下文「XGBoost 模型详解」)

### Step 4: 综合排名

```
final_score = 100/factor_rank + 50/ml_rank + 20*(双重确认加分)
```

- 多因子排名前 50 名进入候选池
- 如果一只股票**同时在多因子 Top20 和 ML Top20**, 额外 +20 分(「双重确认」)
- 按 final_score 降序, 取 top 3

### Step 5: 实时定价 + 整手

- 腾讯行情接口获取实时价格
- 过滤 ST、停牌(成交量=0)
- 资金等分(20000/3 = 6667 元/只), 计算 100 股整数手

## 二、持仓管理

每天先检查现有持仓, 三种卖出信号:

| 触发条件 | 动作 |
|---|---|
| 浮亏 <= **-8%** | 止损卖出 |
| 浮盈 >= **+15%** | 止盈卖出 |
| 持有 >= **20 个交易日** 且涨跌幅 < 3% | 超时调仓(横盘太久换股) |

执行顺序: **先卖后买**。卖出回笼的资金参与新选股。

```
东方财富新闻 → GLM-4-flash 批量标注情绪(-1~+1) → GLM-5 深度研判 → 综合分
```

- 权重: flash 70% + GLM-5 30%

- 在标准模式下会影响 ETF/个股配比(偏空时 80% ETF 防守)

## XGBoost 模型详解

### 学的是什么

**输入**(X): 某只股票在某一天的 19 个因子快照

**输出**(y): 这只股票未来 20 个交易日的真实收益率

```
label = close[T+20] / close[T] - 1
```

本质是一个**回归问题**: 给定当前因子状态, 预测未来 20 日涨跌幅。

### 样本生成: 滚动截面

对每只股票的历史日线(2020 至今):

```
日线: [day0, day1, ... day60, ... day80, ... day100, ...]
          |                    |                    |
       样本1起点            样本2起点            样本3起点
       (每隔20天取一个截面)
```

具体逻辑:

```python
for end_idx in range(60, len(df) - 20, 20):
    window = df[:end_idx+1]      # 截面日以前的全部历史 → 算因子
    fwd = df[end_idx:end_idx+21] # 截面日之后 20 天 → 算真实收益
```

- 每只股票贡献 ~50-70 个样本(取决于历史长度)
- ~1600 只股票 x ~60 样本 = **~10 万条训练数据**
- 每条样本的因子**当场从历史窗口计算**, 不是提前存好的, 确保没有未来信息泄露

### 19 个特征

```
动量(4个):    mom_5d, mom_10d, mom_20d, mom_60d     — 过去 N 日涨幅
波动率(2个):  vol_10d, vol_20d                       — 过去 N 日收益率标准差
换手(3个):    avg_turnover_5d, avg_turnover_20d,     — 平均换手率 + 加速度
              turnover_accel
量价(2个):    vol_price_diverge, volume_surge         — 量价背离 + 放量信号
技术(3个):    ma5_bias, ma10_bias, ma20_bias, rsi_14 — 均线偏离 + RSI
基本面(4个):  pe_ttm, pb, turnover_rate, volume_ratio — 从数据库读当日值
情绪(1个):    sentiment_score                         — 训练时为空, 用中位数填充
```

### 模型参数

```python
XGBRegressor(
    n_estimators=500,          # 最多 500 棵树
    max_depth=4,               # 每棵树最大 4 层(防过拟合)
    learning_rate=0.05,        # 学习率
    subsample=0.8,             # 每棵树随机用 80% 样本
    colsample_bytree=0.8,      # 每棵树随机用 80% 特征
    reg_alpha=0.1,             # L1 正则化
    reg_lambda=1.0,            # L2 正则化
    early_stopping_rounds=20,  # 验证集 20 轮不改善就停
)
```

训练流程:

1. **TimeSeriesSplit 5 折交叉验证** — 严格按时间顺序分折, 防止未来数据泄露, 评估 R²
2. **80/20 时间分割** — 前 80% 训练, 后 20% 做 early stopping 验证集
3. **版本管理** — 新模型 R² > 旧模型才上线, 否则保存为候选不替换

### 预测: 怎么用

```
输入: 今天全市场 ~1600 只候选股的当日因子矩阵
      ↓
      缺失值用截面中位数填充
      ↓
      model.predict(X)
      ↓
输出: 每只股票一个浮点数 = 预测未来 20 日收益率
      ↓
      按 predicted_return 降序排名 → ML排名
      ↓
      与多因子排名综合, 双重确认的优先推荐
```

### R² = 0.0757 意味着什么

模型能解释未来收益率 7.6% 的方差。在 A 股截面预测中:

| R² 范围 | 含义 |
|---|---|
| 0.02 ~ 0.05 | 及格, 有微弱预测力 |
| **0.05 ~ 0.10** | **不错, 排名前 20% 有信息含量(当前水平)** |
| 0.10 ~ 0.20 | 优秀 |
| > 0.50 | 几乎肯定有 bug 或数据泄露 |

0.0757 属于「有用但不神奇」— Top 排名的股票**平均**比随机选的好, 但单只的预测不靠谱。这正是为什么系统用**多因子 + ML 双重确认**而非纯靠模型: 两个独立视角都看好的股票, 可信度更高。
